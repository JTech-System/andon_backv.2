Roles Testing Steps

Role Creation:

  • Create a Role: Create a new role entity with a specific name.
  • Assign Permissions: Assign permissions to the role either by selecting existing permissions or creating new ones. This can be done by updating the permissions array of the role entity.

Role Assignment:

  • Assign to User: Assign roles to users. This could be done by updating the roles array in the User entity.

Role Update:

  • Update Permissions: To update the permissions of a role, you can add or remove permissions from the permissions array. Recalculate the permission bitmask if it's being used.

Permission Checks:

  • Using Permissions Array: Iterate through the permissions array and check if the required permission exists for detailed checks.
